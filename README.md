# GestureWar

**GestureWar** 是一款基于深度学习驱动、支持多人联机的可视频交互枪战程序。本项目结合了 Unity3D 渲染引擎与实时计算机视觉技术，旨在打造一种全新的沉浸式交互体验。

---

## 🚀 项目概览
- **开发周期**：4 周 (暑假实训)
- **团队规模**：5 人
- **核心技术**：
  - **引擎**：Unity3D (C#)
  - **AI/视觉**：Python (MediaPipe/YOLOv5/OpenCV)
  - **通信**：UDP/TCP Socket 进程间通信
  - **网络**：C/S 架构多人联网逻辑

---

## 📅 4周开发计划

### 第一周：架构设计与环境搭建
- **目标**：完成技术选型，搭建前后端通信链路。
- **关键任务**：
  - 初始化 GitHub 协作流程。
  - 搭建 Unity 基础场景与玩家控制器原型。
  - 实现 Python 摄像头采集与初步手势识别 (如：MediaPipe 手部关键点)。
  - 完成 UDP 通信 Demo (Python 发送坐标 -> Unity 接收并移动物体)。

### 第二周：核心机制与深度学习集成
- **目标**：实现稳定的交互控制与游戏逻辑。
- **关键任务**：
  - 开发枪战核心逻辑 (射击、血量、武器系统)。
  - 训练或优化深度学习模型，识别特定手势 (如：握拳射击、平铺移动)。
  - 集成噪声平滑算法 (Kalman 滤波) 减少手势跳变。
  - 实现初步的多人联机同步 (玩家位置同步)。

### 第三周：视频交互与内容填充
- **目标**：完善视觉交互界面与游戏内容。
- **关键任务**：
  - 在 Unity UI 中实时显示摄像头视频流。
  - 增加 AI NPC 敌人系统。
  - 构建复杂的对战场景 (地形、建筑、粒子特效)。
  - 加入音效系统与背景音乐。

### 第四周：联机优化与交付准备
- **目标**：确保流畅运行，完成所有文档。
- **关键任务**：
  - 多人联网压力测试与 Bug 修复。
  - 性能优化 (确保视频处理不卡顿)。
  - 录制演示视频，编写操作手册与技术文档。
  - 最终成果展示。

## 👥 团队分工 (建议)
1. **组长 (1人)**：负责整体架构设计、多人联机后端逻辑、GitHub 维护及进度管理。
2. **Unity 开发 A (1人)**：负责玩家控制、武器系统、UI 界面及小地图。
3. **Unity 开发 B (1人)**：负责场景构建、NPC AI、粒子特效及音效。
4. **AI 算法工程师 (1人)**：负责 Python 端手势识别、深度学习模型训练及数据通信。
5. **测试与文档 (1人)**：负责功能测试、Kalman 滤波优化方案、技术文档及演示视频制作。

---

## 🛠️ 技术选型建议
- **手势识别**：首选 **MediaPipe** (实时性极佳) 或 **YOLOv5** (针对特定物体/姿态识别)。
- **通信协议**：使用 **UDP** 进行高频位置同步，**TCP** 进行状态切换 (如开始游戏、重置)。
- **平滑处理**：必须加入滤波算法，否则摄像头采集的坐标跳变会严重影响游戏体验。
